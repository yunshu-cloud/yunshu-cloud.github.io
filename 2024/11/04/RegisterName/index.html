RegisterName

When it comes to operating systems, especially for x86-64 architecture,we often encounter various types of registers, and their names can be confusing.Today, I’ll explain them in detail.

Registers in the x86-64 Architecture

Register Full Name Usage
RAX Accumulator Register be used for arithmetic and logical operations.
RBX Base Register be used as a base pointer for memory access.
RCX Counter Register be used as a loop counter in LOOP and shift/rotate instructions.
RDX Data Register be used in multiplication and division operations
RSI Source Register be used as a source pointer for string and memory operations
RDI Destination Register be used as the destination pointer in string operation
RBP Base Pointer Register used as a frame pointer for stack frames in function calls.
RSP Stack Register holds the address of the top of the stack
R9 Additional General-Purpose Register provides extra general-purpose storage
… … …
R15 Additional General-Purpose Register provides extra general-purpose storage

Register History

  • On a 16-bit machine, there are 16 bits (0-15) available for storage. The AL register represents the lower 8 bits (0-7), while AH represents the upper 8 bits (8-15). The AX register represents all 16 bits combined.
  • On a 32-bit machine, registers are prefixed with E (for “Extended”) to indicate a 32-bit extension of the original 16-bit registers.
  • On a 64-bit machine, registers are prefixed with R (for “Register”) to signify the 64-bit version of the register.

Summary of Naming

The first eight registers (RAX, RBX, RCX, RDX, RSI, RDI, RBP, and RSP) retain their original x86 names due to their historic roles in assembly programming and compatibility requirements. The names of the newer registers (R8 to R15) simply follow a numerical sequence (R8, R9, etc.), reflecting their additional, general-purpose nature without specific historical functions. These additional registers allow x86-64 to handle more data in parallel, which is essential for modern, complex applications.
